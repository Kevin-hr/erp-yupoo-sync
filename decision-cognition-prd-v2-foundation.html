Decision Cognition System v2.0

Phase 1: Foundation Router

可视化 PRD 幻灯片 - 面向产品/管理的可分享简报

核心要点

  • 目标：在极短时间内对复杂决策进行分类与初步取舍
  • 实现：ROI 红线、场景分类、主要矛盾提取、快速路径与熔断保护
  • 产出：可视化的决策路径与落地行动建议

背景与机会

  • 网红 ROI 追踪场景复杂，决策需跨领域视角（资源、风险、机会成本）
  • 现状痛点：信息噪声大、单点决策缓慢、缺乏统一的路由与保护机制
  • 机会：通过阶段性路由与鲁棒保护，提升决策速度与一致性

高层图景

愿景与目标

  • 愿景：将复杂决策系统化、可追溯、具备自我保护能力的智能决策助手
  • 目标（Phase 1）：Foundation Router 的准确分类、快速决策、以及安全的执行路径
  • 目标指标：
    - 场景分类在 2 秒内返回
    - ROI 负面的即时拦截
    - 快速路径在 2 分钟内完成“快速决策”

范围定义

  • In-Scope：类型系统、ROI 守卫、场景分类、主要矛盾提取、路由规则、CLI入口、基础工作流（Phase 1）
  • Out-Of-Scope：多语言国际化、移动端推送、独立的 UI 界面

系统总览

  • Input: 自然语言描述的决策场景
  • Router: ROI 检测 → 场景分类 (TRIVIAL/REVERSIBLE/MAJOR/INNOVATIVE/EMOTIONAL) → 主要矛盾提取 → Agent 列表生成
  • Workflow (INFRA-02): 逐步调度代理，Phase 1 为顺序执行
  • Guard: CircuitBreaker（最大跳数 5）和 SLA（2 分钟）
  • Output: 结论、行动建议、以及日志记录

关键特性 (Phase 1)

  • ROI 红线：负 ROI 时拦截，确保资源不被错误投入
  • 场景分类：基于规则的 CORE-01，覆盖 trivial, reversible, major, innovative, emotional
  • 主要矛盾提取：CORE-02，快速识别资源、风险、选择、增长等矛盾
  • 快速路径（2 分钟 SLA）: trivial/reversible 走 fast path，无需完整 Agent 链
  • 熔断器：最大跳数 5，超过后返回 circuit_open 错误
  • Sequential Dispatch：Phase 1 为顺序派发（INFRA-02）

数据模型 (核心类型)

SceneType 枚举

  • TRIVIAL, REVERSIBLE, MAJOR, INNOVATIVE, EMOTIONAL, BLOCKED

RouterResult 字段

  • scenario_type, main_conflict, complexity, emotion_state
  • options_clear, recommended_agents, reason, key_questions
  • hop_count, roi_blocked

用户旅程（Actors）

  • 产品经理/分析师：提交场景描述 → 接收分类与初步冲突 → 看到推荐的行动路径
  • 工程师/实现者：通过 CLI/API 验证路由和输出结构
  • 运营/数据团队：查看日志（JSONL），评估决策质量与可追踪性

非功能性要求

  • Performance：场景分类应在 2 秒内完成（无LLM调用时）
  • Reliability：熔断器在达到跳数上限时触发 circuit_open
  • Security & Validation：输入校验、避免信息泄露，日志记录审计
  • Testing：阶段性单元测试与集成测试的覆盖率提升

里程碑与路线图 (Phase 1 → Phase 4)

  • Phase 1（已完成）: Foundation Router + CLI + Sequential Dispatch
  • Phase 2: Parallel Dispatch 設計与基础实现
  • Phase 3: 实现真实 Agent 调用（wise/bias/reverse/second/first principle）
  • Phase 4: End-to-end UAT 与 生产就绪

风险与缓解

  • 依赖缺失/测试覆蓋不足 → 立即完善测试基线与依赖清单
  • Emotion detection 未实现 → 先实现简单关键词触发再迭代
  • 导入路径/导出接口的集成风险 → 提前实现 __init__ 暴露并统一导出

验证与关键指标 (KPIs)

  • 分类延迟：目标 < 2 秒
  • ROI 拦截：Negative ROI 场景立即 BLOCKED
  • 熔断：跳数达 5 时 circuit_open
  • 快速路径：Trivial/Reversible 跳过 LLm 调用，给出明确建议
  • 日志：JSONL 日志持续增长，便于审计和回放

附录 / 术语

  • ROI: 投资回报率
  • BLOCKED: ROI 红线触发，决策阻断
  • EMOTIONAL: 情绪驱动的场景
  • SLA: 服务水平协议（2 分钟）
